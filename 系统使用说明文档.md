## 1. 系统简介

### 1.1 系统概述

学生成绩管理系统是一个用于管理学生选课和成绩的Web应用系统。系统支持学生选课、教师录入成绩、管理员管理课程等功能。

### 1.2 用户角色

系统支持三种用户角色：

- **学生（student）**：可以选课、查询成绩
- **教师（teacher）**：可以录入成绩、管理课程
- **管理员（admin）**：可以管理用户、课程等所有功能

### 1.3 主要功能

- ✅ 用户注册和登录
- ✅ 课程信息管理
- ✅ 学生选课和退课
- ✅ 成绩录入和查询

## 3. 快速开始

### 3.1 访问系统

1. 打开浏览器
2. 打开登录页面文件：`登陆界面.html`
3. 如果已有账号，直接登录；如果没有账号，请先注册

### 3.2 首次使用

**步骤1：注册账号**

1. 点击登录页面的"注册账号"按钮
2. 进入注册页面
3. 填写以下信息：
   - **用户名**：输入您的用户名（必填）
   - **密码**：输入您的密码（必填）
   - **角色**：选择您的角色（student/teacher/admin）
4. 点击"Sign in"按钮提交注册
5. 看到"用户添加成功"提示后，注册完成

**步骤2：登录系统**

1. 返回登录页面
2. 输入注册时的用户名和密码
3. 点击"Login"按钮
4. 看到"登录成功"提示后，即可使用系统

## 4. 功能使用指南

### 4.1 用户注册功能

#### 4.1.1 操作步骤

1. **打开注册页面**
   - 在登录页面点击"注册账号"按钮
   - 或直接打开`注册界面.html`文件

2. **填写注册信息**
   ```
   用户名：输入您要使用的用户名（例如：zhangsan）
   密码：输入您的密码（例如：123456）
   ```

3. **提交注册**
   - 点击"Sign in"按钮
   - 系统会自动发送注册请求

4. **查看结果**
   - 成功：页面会弹出"用户添加成功"提示
   - 失败：页面会显示相应错误信息

#### 4.1.2 注意事项

- ⚠️ 用户名不能为空
- ⚠️ 密码不能为空
- ⚠️ 用户名在系统中必须唯一，如果用户名已存在，注册会失败
- ⚠️ 注册成功后，请记住您的用户名和密码

### 4.2 用户登录功能

#### 4.2.1 操作步骤

1. **打开登录页面**
   - 打开`登陆界面.html`文件

2. **输入登录信息**
   ```
   用户名：输入您注册时的用户名
   密码：输入您注册时的密码
   ```

3. **点击登录**
   - 点击"Login"按钮
   - 系统会验证您的用户名和密码

4. **查看结果**
   - **成功**：页面会弹出"登录成功"提示，可以继续使用系统
   - **失败**：页面会弹出"用户名或密码错误"提示，请检查输入是否正确

#### 4.2.2 常见问题

**Q: 忘记密码怎么办？**
- A: 目前系统暂不支持密码找回功能，请联系管理员重置密码。

**Q: 登录后如何退出？**
- A: 关闭浏览器标签页即可退出登录（当前版本为无状态登录）。

### 4.3 课程管理功能

#### 4.3.1 查看所有课程

**适用角色**：所有用户

**操作步骤**：

1. 使用API工具（如Postman）或前端页面发送请求
2. 请求地址：`GET http://localhost:8089/courseManagement/getAllCourse`
3. 系统返回所有课程列表（JSON格式）

**返回示例**：
```json
[
  {
    "id": 1,
    "name": "高等数学",
    "credit": 4.0,
    "teacherId": 101,
    "userId": 1
  },
  {
    "id": 2,
    "name": "Java程序设计",
    "credit": 3.0,
    "teacherId": 102,
    "userId": 1
  }
]
```

#### 4.3.2 添加课程

**适用角色**：教师、管理员

**操作步骤**：

1. 准备课程信息（JSON格式）：
```json
{
  "name": "数据库系统原理",
  "credit": 3.5,
  "teacherId": 103,
  "userId": 1
}
```

2. 发送POST请求：
   - 地址：`POST http://localhost:8089/courseManagement/addCourse`
   - 请求头：`Content-Type: application/json`
   - 请求体：上述JSON数据

3. 查看结果：
   - 成功：返回"课程增加成功"
   - 失败：返回相应错误信息

#### 4.3.3 更新课程信息

**适用角色**：教师、管理员

**操作步骤**：

1. 准备更新的课程信息（必须包含课程ID）：
```json
{
  "id": 1,
  "name": "高等数学（上）",
  "credit": 4.5,
  "teacherId": 101,
  "userId": 1
}
```

2. 发送PUT请求：
   - 地址：`PUT http://localhost:8089/courseManagement/updateCourse`
   - 请求头：`Content-Type: application/json`
   - 请求体：上述JSON数据

3. 查看结果：
   - 成功：返回"课程修改成功"

#### 4.3.4 删除课程

**适用角色**：管理员

**操作步骤**：

1. 发送DELETE请求：
   - 地址：`DELETE http://localhost:8089/courseManagement/deleteCourse/{课程名称}`
   - 示例：`DELETE http://localhost:8089/courseManagement/deleteCourse/高等数学`

2. 查看结果：
   - 成功：返回"课程删除成功"

**注意事项**：
- ⚠️ 删除课程前，请确保没有学生选择该课程
- ⚠️ 删除操作不可恢复，请谨慎操作

### 4.4 选课管理功能

#### 4.4.1 学生选课

**适用角色**：学生

**操作步骤**：

1. 确认要选择的课程ID和学生ID
2. 发送PUT请求：
   - 地址：`PUT http://localhost:8089/courseSelect/selectOneCourse`
   - 参数：
     - `studentId`：学生ID（必填）
     - `courseId`：课程ID（必填）
   - 示例：`PUT http://localhost:8089/courseSelect/selectOneCourse?studentId=1&courseId=1`

3. 查看结果：
   - 成功：返回"选课成功"
   - 失败：返回"参数错误：学生ID和课程ID不能为空"

**注意事项**：
- ⚠️ 每个学生不能重复选择同一门课程
- ⚠️ 选课前请确认课程信息正确

#### 4.4.2 学生退课

**适用角色**：学生

**操作步骤**：

1. 确认要退选的课程ID和学生ID
2. 发送DELETE请求：
   - 地址：`DELETE http://localhost:8089/courseSelect/deleteCourseSelection/{student_id}/{course_id}`
   - 示例：`DELETE http://localhost:8089/courseSelect/deleteCourseSelection/1/1`

3. 查看结果：
   - 成功：返回"课程删除完毕"

**注意事项**：
- ⚠️ 退课后，该课程的成绩记录也会被删除
- ⚠️ 请谨慎操作，避免误退课程

### 4.5 成绩管理功能

#### 4.5.1 录入成绩

**适用角色**：教师、管理员

**操作步骤**：

1. 确认学生ID、课程ID和成绩
2. 发送POST请求：
   - 地址：`POST http://localhost:8089/scoreManagement/addScore`
   - 参数：
     - `StudentId`：学生ID（必填）
     - `CourseId`：课程ID（必填）
     - `score`：成绩，范围0-100（必填）
   - 示例：`POST http://localhost:8089/scoreManagement/addScore?StudentId=1&CourseId=1&score=85`

3. 查看结果：
   - 成功：返回"成绩写入完成"
   - 失败：
     - 成绩不在0-100范围内：返回"成绩必须在0-100之间"
     - 未找到选课记录：操作失败（无提示）

**注意事项**：
- ⚠️ 成绩必须在0-100之间
- ⚠️ 只能为已选课的学生录入成绩
- ⚠️ 可以多次修改成绩，最后一次修改会覆盖之前的成绩

#### 4.5.2 查询成绩

**适用角色**：所有用户

**操作步骤**：

1. 确认要查询的学生ID和课程ID
2. 发送GET请求：
   - 地址：`GET http://localhost:8089/scoreManagement/getScore`
   - 参数：
     - `StudentId`：学生ID（必填）
     - `CourseId`：课程ID（必填）
   - 示例：`GET http://localhost:8089/scoreManagement/getScore?StudentId=1&CourseId=1`

3. 查看结果：
   - 成功：返回"成绩：85"（示例）
   - 未找到记录：返回"未找到对应的成绩记录"

**成绩说明**：
- 成绩范围：0-100分
- 如果返回-1，表示该课程成绩尚未录入

## 5. 常见问题

### 5.1 登录相关问题

**Q1: 登录时提示"用户名或密码错误"怎么办？**

**A:** 请检查以下几点：
1. 确认用户名和密码输入正确（注意大小写）
2. 确认账号已成功注册
3. 如果忘记密码，请联系管理员

**Q2: 注册时提示"用户添加成功"但无法登录？**

**A:** 可能的原因：
1. 注册时角色信息未正确设置
2. 数据库连接问题
3. 请尝试重新注册或联系管理员

### 5.2 选课相关问题

**Q1: 选课时提示"参数错误"？**

**A:** 请检查：
1. 学生ID和课程ID是否正确
2. 参数是否完整（studentId和courseId都不能为空）
3. 学生ID和课程ID是否为有效的数字

**Q2: 可以重复选同一门课程吗？**

**A:** 不可以。系统不允许学生重复选择同一门课程。如果需要重新选课，请先退选再重新选择。

### 5.3 成绩相关问题

**Q1: 录入成绩时提示"成绩必须在0-100之间"？**

**A:** 请确保：
1. 成绩值在0-100范围内
2. 输入的是数字，不是文本
3. 没有输入负数或超过100的数值

**Q2: 查询成绩时显示"未找到对应的成绩记录"？**

**A:** 可能的原因：
1. 该学生未选择该课程
2. 成绩尚未录入（成绩为-1）
3. 学生ID或课程ID输入错误

### 5.4 系统使用问题

**Q1: 页面无法打开或显示异常？**

**A:** 请检查：
1. 浏览器是否支持JavaScript
2. 后端服务是否已启动（默认端口8089）
3. 网络连接是否正常
4. 尝试刷新页面或清除浏览器缓存

**Q2: 请求失败，提示网络错误？**

**A:** 请检查：
1. 后端服务是否正在运行
2. 后端地址是否正确（默认：http://localhost:8089）
3. 防火墙是否阻止了连接
4. 浏览器控制台是否有详细错误信息

**Q3: 如何联系管理员？**

**A:** 请联系系统管理员获取帮助，或查看项目文档中的联系方式。

## 6. 注意事项

### 6.1 数据安全

- 🔒 **密码安全**：请妥善保管您的密码，不要将密码告诉他人
- 🔒 **账号安全**：使用完毕后，请关闭浏览器标签页
- 🔒 **数据备份**：重要数据请自行备份，系统不提供自动备份功能

### 6.2 操作规范

- ✅ **规范操作**：请按照系统提示进行操作，不要随意修改URL参数
- ✅ **数据准确性**：录入数据时，请仔细核对信息的准确性
- ✅ **及时保存**：完成操作后，请确认系统返回的成功提示

### 6.3 系统限制

- ⚠️ **成绩范围**：成绩必须在0-100之间
- ⚠️ **选课限制**：每个学生不能重复选择同一门课程
- ⚠️ **权限限制**：不同角色的用户有不同的操作权限，请按照您的角色进行操作

### 6.4 浏览器兼容性

- 🌐 **推荐浏览器**：Chrome、Firefox、Edge最新版本
- 🌐 **JavaScript**：必须启用JavaScript功能
- 🌐 **Cookie**：建议启用Cookie功能（未来版本可能需要）

### 6.5 网络要求

- 📡 **网络连接**：需要稳定的网络连接
- 📡 **服务器地址**：默认后端地址为`http://localhost:8089`
- 📡 **跨域访问**：系统已配置跨域支持，可以正常访问

## 10. 附录

### 10.1 API接口快速参考

| 功能 | 方法 | 路径 | 参数 |
|------|------|------|------|
| 用户注册 | POST | `/user/register` | JSON: username, password, role |
| 用户登录 | GET | `/user/login` | username, password |
| 删除用户 | DELETE | `/user/delete/{username}` | path: username |
| 获取所有课程 | GET | `/courseManagement/getAllCourse` | 无 |
| 添加课程 | POST | `/courseManagement/addCourse` | JSON: name, credit, teacherId, userId |
| 更新课程 | PUT | `/courseManagement/updateCourse` | JSON: id, name, credit, teacherId, userId |
| 删除课程 | DELETE | `/courseManagement/deleteCourse/{name}` | path: name |
| 选课 | PUT | `/courseSelect/selectOneCourse` | studentId, courseId |
| 退课 | DELETE | `/courseSelect/deleteCourseSelection/{student_id}/{course_id}` | path: student_id, course_id |
| 录入成绩 | POST | `/scoreManagement/addScore` | StudentId, CourseId, score |
| 查询成绩 | GET | `/scoreManagement/getScore` | StudentId, CourseId |

### 10.2 数据格式说明

**用户注册JSON格式**：
```json
{
  "username": "用户名",
  "password": "密码",
  "role": "student|teacher|admin"
}
```

**课程信息JSON格式**：
```json
{
  "id": 1,
  "name": "课程名称",
  "credit": 3.5,
  "teacherId": 101,
  "userId": 1
}
```
